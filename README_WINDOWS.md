# Замечания по использованию в Windows

## Возможные проблемы с сохранением файлов

При использовании программы в Windows могут возникать проблемы с сохранением файлов на файловую систему. Это связано с различиями в обработке путей между операционными системами.

### Основные причины проблем:

1. **Различия в форматах путей**: Windows использует обратные слэши (`\`) и буквы дисков (C:\), в то время как Unix-подобные системы используют прямые слэши (/).

2. **Проблемы с канонизацией путей**: Функция `canonicalize()` в Rust работает по-разному на разных платформах, особенно при работе с путями на разных дисках в Windows.

3. **Права доступа к файловой системе**: В Windows могут быть ограничения на запись в определенные каталоги без прав администратора.

### Рекомендации по использованию в Windows:

1. **Используйте абсолютные пути**: Вместо относительных путей используйте абсолютные, например:
   ```
   rustore_apk_downloader.exe com.example.app C:\Downloads
   ```

2. **Убедитесь в наличии прав на запись**: Убедитесь, что у вас есть права на запись в указанный каталог загрузки.

3. **Избегайте системных каталогов**: Не пытайтесь сохранять файлы в системные каталоги, такие как `C:\Windows` или `C:\Program Files`.

4. **Используйте каталоги пользователя**: Рекомендуется использовать каталоги в пользовательской папке, например:
   ```
   rustore_apk_downloader.exe com.example.app %USERPROFILE%\Downloads
   ```

### Отладка проблем:

Если у вас возникают проблемы с сохранением файлов в Windows, запустите программу с переменной окружения `RUST_LOG=debug` для получения подробной информации:

```
set RUST_LOG=debug
rustore_apk_downloader.exe com.example.app C:\Downloads
```

Это позволит увидеть детали процесса обработки путей и сохранения файлов, что поможет определить точное место возникновения проблемы.
